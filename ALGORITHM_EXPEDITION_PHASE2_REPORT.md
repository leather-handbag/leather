## 3. 账号名字安全

新账号：

- 邮箱注册与 GitHub OAuth 均使用邮箱 `@` 前缀。
- 进行 NFKC 规范化、控制字符清理、30 字截断和敏感内容检查。
- 不安全初始前缀回退为 `user_<UUID8>`，不会因为邮箱前缀而封禁注册。
- 现有用户资料未批量覆盖。

资料修改：

- `update_my_profile` 返回 `{ok,banned,reason,profile}`。
- 名字命中敏感词、脚本、导流、手机号、不可见字符或长度攻击时，不写入同次提交的名字、ID 和简介。
- 普通用户被原子封禁，资料重置为“已封禁用户”、安全 ID、空简介和无头像。
- 违规事件只记录类别，不在普通页面公开原始名字。
- 站长防误封规则仍有效。

远程回滚事务已验证：违规名字会返回 `banned=true`，提交的 handle/bio 均未部分写入；事务结束后站长角色、保留 ID 和封禁状态恢复正常。

## 5. 算法地图与能力直达

- 地图总数：7。
- 区域总数：41。
- 区域标题已由幻想地名改为“二分与双指针”“网络流与匹配”“多项式与线性代数”等算法类别。
- 每个区域展示有效 AC、证据目标、上段证据、子技能覆盖、训练日、置信度及评分解释。
- 锁定地图也展示算法列表和两种解锁方式。

能力估计只使用 Codeforces/AtCoder 中有标准难度、已 AC、镜像去重后的独立题目。样本公式为：

`k = min(20, max(5, ceil(n * 25%)))`

直达解锁记录 `reason=ability_average`、均值、样本数、门槛和模型版本；只开放地图，不伪造前置区域 100%，也不授予地图制霸。线上已有 1 份能力估计，并由真实数据产生 5 个永久直达解锁，证明迁移重算路径有效。

探险榜现优先按制霸地图数排序，开放地图数仅作次级指标。

## 6. 月度学习报告

- Cron：`20 16 * * *`（UTC），即北京时间每天 00:20 检查。
- 仅北京时间每月 2 日生成上一个自然月。
- 唯一键：`(user_id, report_month)`。

远程回滚测试已验证：

- 同月调用两次返回同一报告 ID。
- 数据库仅保留一行报告。
- 41 个区域快照完整。

## 10. 已知风险与后续项

1. Supabase Advisor 仍将 `public_profile_stats` 和 `post_engagement` 标记为 `security definer view`。这两个 View 是既有公共裁剪 DTO，直接改为 `security_invoker` 会因底层敏感表权限导致公开主页和文章统计失效。应在单独迁移中改为专用 RPC 或公共投影表后再消除此警告。
2. Advisor 对旧 RLS 中逐行调用 `auth.uid()` 给出性能建议。当前数据量下不影响功能，但可集中改为 `(select auth.uid())`，避免混入本阶段行为修改。
3. 公开头像存储桶仍有可列目录策略警告。对象 URL 本身需要公开，但列表策略可以在单独存储权限迁移中收紧。
4. 前端生产包已构建；若仓库未提交并推送，GitHub Pages 不会自动获得本阶段界面更新。数据库和 Edge 已经线上生效。
