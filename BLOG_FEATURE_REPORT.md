# 博客、社区与社交功能报告

博客与评论已经迁移到 Supabase 数据层：

- 私有文章仅作者和管理员/站长可读，公开文章对访客开放。
- 我的文章搜索页与独立编辑页已经分开；编辑与 Markdown/LaTeX 预览固定双栏同步。
- 博客自动保存改为开关，开启后固定每 30 分钟保存；手动和自动保存都会产生版本，最多保留 100 个，可一键恢复。
- 文章作者只能修改自己的文章；评论作者只能修改或删除自己的评论。
- 管理员只能删除普通用户内容，不能修改作者内容，也不能处理站长或其他管理员。
- Markdown、LaTeX、安全链接和 HTML 转义策略保持不变。
- 前端敏感内容提示之外，数据库触发器还会执行最终审核、删除和封禁。
- 原留言模块已升级为独立讨论区，分为灌水区、学术区、站务区；支持回复、`@用户ID`、作者自删和管理员删除。
- 文章支持点赞和收藏；每个账号有默认收藏夹，也可新建、重命名、删除和移动收藏。
- 用户可以关注他人，个人主页显示关注/粉丝数量与按签到、博客、讨论、粉丝解锁的成就徽章。
- 铃铛通知分为评论回复、关注和系统通知；头像审核结果与成就解锁由服务端生成系统通知。

数据库表、RLS、审核触发器和部署操作见 [SUPABASE_SETUP_REPORT.md](SUPABASE_SETUP_REPORT.md)。旧版 `localStorage` 博客不会自动上传，避免在用户不知情时公开本地草稿；需要迁移时应先人工检查文章可见性和敏感内容。
