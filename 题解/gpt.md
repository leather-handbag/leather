# Leather 交付说明

已在 `leather` 仓库内完成面向信息学竞赛生的纯静态代码辅助站。

## 已完成

- 首页：介绍模板库、对拍和任务导图，并理性说明本地优先、免费部署、无后端依赖等特点与相应限制。
- 模板库：关键字定位独立代码库，密码经 PBKDF2 派生密钥，使用 AES-256-GCM 加密后写入 localStorage；支持分区、标签页、标签、语言筛选、版本快照、逐行 Diff、旧版本恢复、密文导入导出。
- 对拍：JavaScript 可在 Web Worker 中一键对拍，支持组数、固定种子、运行进度、超时中止、首个差异现场；内置随机数组、整数和连通图模板。
- C++14 对拍：可下载 PowerShell 本地对拍脚本，自动调用 `g++` 编译正解、暴力和生成器并逐组比较。
- 任务导图：任务按层分组，上层优先级更高；支持任意层添加任务、跨层拖动、层级上下移动、编辑、删除、完成状态和总体进度。
- 部署：已添加 `.nojekyll`，使用 Hash 路由，可直接通过 GitHub Pages 从仓库根目录发布。

## 已验证

- Edge 浏览器自动化烟雾测试通过：导图渲染、新增任务、在线对拍、加密库创建、版本保存和模板 ID 稳定性。
- 网页生成的 C++14 对拍脚本已用本机 `g++` 编译并跑完 5 组测试，全部通过。
- 390px 移动端视口检查通过，页面宽度与文档宽度一致，无横向溢出。

## 需要仓库所有者操作

1. 将改动提交并推送到 GitHub 的 `leather` 仓库。
2. 在仓库 Settings → Pages 中选择 Deploy from a branch，发布默认分支的 `/ (root)`。
3. 如需开源协作，请自行选择并补充 LICENSE。

GitHub Pages 无法直接在服务器端编译 C++，因此网页内的一键在线执行使用 JavaScript；C++14 使用网站生成的本地脚本完成。本地代码库不会自动跨设备同步，建议定期导出密文备份；密码无法找回。
